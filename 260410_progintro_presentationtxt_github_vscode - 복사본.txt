[슬라이드 4] 협업의 중심부 GitHub

GitHub을 단순히 '코드를 올려두는 클라우드 드라이브' 정도로 오해하기 쉽습니다.
하지만 실무에서 GitHub은 완벽한 '협업의 중심지'입니다.
화면에 보이시는 것처럼 누가 어떤 버그를 고칠지 논의하는 '이슈(Issues)' 기능,
내 코드를 메인 프로젝트에 합쳐도 될지 팀원들에게 코드 리뷰를 요청하는 '풀 리퀘스트(Pull Requests)',
그리고 진행 상황을 칸반 보드 형태로 한눈에 관리하는 '프로젝트(Projects)' 기능까지 제공합니다.

즉, 개발과 관련된 모든 커뮤니케이션이 깃허브라는 공간 안에서 이루어진다고 보시면 되겠습니다.
